Чего “не хватает” для 3x-ui/Xray: нет кода, который создаёт/обновляет пользователя в 3x-ui и выдаёт VLESS-REALITY ссылку. Сейчас у вас VpnAPI хранит “клиента”, а бот показывает/принимает алерты — но “выдать ключ” через 3x-ui ещё не реализовано.

Схема A (рекомендую): Reality на 443, Nginx не трогает 443
Xray (VLESS‑REALITY) слушает 443/tcp на сервере.
Nginx для панели/API вешаете на другой порт, например:
панель: https://alexsurfervpn.space:8443 → прокси на 127.0.0.1:2053
(опционально) API: https://alexsurfervpn.space:9443 → прокси на 127.0.0.1:8080
Плюс: максимально “как задумано” для Reality. Минус: панель/апи будут на нестандартных портах (их можно закрыть по IP).

Интеграция “выдать ключ”:
VpnAPI остаётся источником правды (кто активен, лимиты, срок).
Бот получает команду “выдать/продлить” → вызывает VpnAPI → дальше нужен модуль, который дергает 3x-ui API и получает VLESS‑REALITY URI.
